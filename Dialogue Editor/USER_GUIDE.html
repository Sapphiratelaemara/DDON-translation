Installation & Setup

The Dialogue Editor requires Python 3.x to run. The application will automatically check for and install required dependencies on first launch.

Required Dependencies:

  • eel - Python-JavaScript bridge for the web interface
  • requests - HTTP library for API calls
  • msgpack - Binary serialization for efficient data storage
  • jamdict - Japanese dictionary integration
Note: On first launch, you will be prompted to install missing dependencies automatically. Simply type 'y' to confirm installation.

Configuration Files:

  • keys.json - API keys for DeepL and OpenRouter (auto-created if missing, stored in root directory)
  • config/<lang>/formatter_config.json - Main application settings (per-language)
  • config/<lang>/user_settings.json - User preferences including sync settings, paths, theme (per-language)
  • config/<lang>/memory.json - Learned fixes and archetype assignments (per-language)
  • config/<lang>/archetypes.json - Character archetypes (synced to GitHub)
  • config/<lang>/dd1_vocab.json - DD1 vocabulary (synced to GitHub)
  • config/<lang>/other_vocab.json - Non-DD1 vocabulary (synced to GitHub)
  • config/<lang>/anach_definitions.json - Anachronism definitions (synced to GitHub)
  • config/<lang>/archaic_examples.json - Archaic word examples (synced to GitHub)
  • config/<lang>/translation_memory.json - Translation memory entries (synced to GitHub, with hash-based entry IDs)
  • config/<lang>/cache.json - API response cache (7-day TTL, per-language)
  • config/<lang>/prefetch_cache.json - Prefetch cache for performance (7-day TTL, per-language)
  • config/<lang>/review_queues_cache.json - Review queues (per-language, not synced)
  • config/<lang>/review_items_cache.json - Manual translation queue (per-language, not synced)

Main Editor Interface

The main editor is where you translate and edit dialogue lines. The interface is divided into several key areas:

Figure 1: Main Editor Interface

Interface Components:

  • Source: Displays the original source text to be translated
  • Translation: Where you edit the translated text
  • Right Sidebar: Toggle CONTEXT, HISTORY, COMMENTS, and AI_ASSISTANT panels using buttons in the header. Shows glossary suggestions, anachronism warnings, translation history, reviewer comments, AI chat interface, and archetype notes
  • Gloss Window: Displays morpheme analysis and linguistic breakdown of source text with parts of speech and dictionary lookups
  • Context Flow: Shows previous and next dialogue lines for context
  • AI Assistant: AI-powered chat interface for translation and editing assistance with conversation history
  • Navigation Controls: Move between items in your queue
  • Save Button: Save your changes to translation history (and optionally to CSV)

Translation History

The editor maintains a translation history for each dialogue entry, tracking all changes made by different reviewers.

  • History Panel: View all translation changes for the current entry, including who made the change (based on sync nickname), timestamp, and the translation text
  • Save to History: Click the Save button or press Ctrl+Enter to save your translation to history (does not write to CSV until explicitly approved)
  • History Persistence: Translation history is synced to GitHub for collaboration with other reviewers

Comments Panel

The Comments panel allows reviewers to communicate about specific translation entries.

  • Add Comment: Enter text in the comment input field and click the Add Comment button to post a comment (max 5000 characters, warning at 4000)
  • View Comments: All comments for the current entry are displayed with the reviewer's name (sync nickname) and timestamp
  • Communication: Use comments to discuss translation issues, suggest improvements, or coordinate with other reviewers
  • Comment Persistence: Comments are synced to GitHub for collaboration with other reviewers

Approve/Reject & Force Save

The editor includes approval workflow features to manage translation quality and CSV writes.

  • Approve Button: Saves the current translation to the CSV file and marks it as approved. This writes the translation to disk and updates the entry status to "approved" in the translation history.
  • Reject Button: Marks the current translation as rejected with a reason. The translation is not written to CSV. Useful for flagging translations that need improvement.
  • Auto-Approve Toggle: When enabled, automatically approves the translation after saving to history. This combines the Save and Approve actions into one step, writing directly to CSV.
  • FORCE_SAVE: When character limits are exceeded (lines too long), the system will block the save. Use FORCE_SAVE to override this check and save anyway (use with caution).
  • Unapproved Entries Queue: View all entries that have been saved to history but not yet approved. This queue shows comment count for each entry.
  • Status Badges: Each translation entry shows its current status: APPROVED, REJECTED, or UNAPPROVED based on approval history.
Tip: Use Ctrl+Z to undo changes in the translation editor.

Settings Configuration

Access the settings panel by clicking the cogwheel icon in the upper right corner to configure file paths, translation options, and custom rules.

Figure 2: Settings Panel

Settings Sections:

  • Language: Select the current language to work with. Each language has its own configuration directory (config/<lang>/) with separate settings files. Switching languages loads the appropriate configuration for that language.
  • GitHub Sync: Configure GitHub repository and token for syncing translation data, archetypes, vocabulary, anachronism definitions, and translation memory. Set your sync nickname to identify your contributions. Enable auto-sync to automatically push/pull on save/load. Sync uses hash-based entry IDs to group translations of the same source text. Conflicts are automatically resolved by keeping both versions with timestamp suffixes.
  • API Keys: Configure DeepL and OpenRouter API keys for translation services
  • Path Mappings: Configure paths to bible (.txt/.log), glossary (.csv), and assets directory
  • Tag Mappings: Define custom tag lengths for automatic line calculations and in-game preview
  • Limit Presets: Configure character limit and wall of text (max lines) presets
  • Replace Rules: Configure regex replacement rules for automated editing during scan
  • Archetypes: Manage character archetypes and their speech patterns
  • AI Prompts: Configure the AI system prompt (used for all AI interactions) and button prompts (Translate, Rephrase, Archaize, Check) for AI assistant. Use {text} placeholder in button prompts for the text to process
  • Regex Sandbox: Test regex patterns and replacements
Note: Changes to settings are saved automatically when you click the save button for each section.

Batch Processing

Batch processing allows you to scan multiple files and apply automated translation rules across your project.

Figure 3: Batch Processing Interface

Dashboard Features:

  • WATCHED_FOLDERS: Configure folders to scan for CSV files
  • ENTRY_KEYS: Set entry keys (e.g., TUTORIAL_STEP) to filter which entries to scan
  • EXECUTE BATCH SCAN: Scan configured folders and populate translation queue
  • PREVIEW_MODE: Toggle preview mode (doesn't write to CSV)
  • CHAR_PRESET & WALL_PRESET: Select character and wall of text presets for automatic linebreaking
  • TRANSLATE CSV: Load a CSV file into the translation queue for editing
Tip: Use batch processing to quickly organize and scan multiple CSV files for translation work.

Lore & Glossary Features

The right sidebar provides context from the game's lore and a glossary of terms to help maintain consistency.

Figure 4: Lore & Glossary Sidebar

Lore Features:

  • Legacy Markers: Items marked with ★ indicate terms used in previous or related works
  • Archetype Notes: Speaker-specific notes based on assigned character archetype

Glossary Features:

  • Term Mapping: Maps source language terms to target language translations
  • Additional Suggestions: Provides alternative translations
  • Text Scanning: Scans source text to find glossary term matches
  • Gloss Integration: Highlights project-specific terms in the gloss window

Anachronism Detection

The editor automatically detects anachronisms (modern words that don't fit the fantasy setting) and highlights them.

Figure 5: Anachronism Highlighting in Text
Figure 6: Anachronism Detail Modal

Anachronism Features:

  • In-Universe Toggle: Enable the in-universe language toggle in the editor to apply anachronism detection and highlighting. When disabled, anachronism features are skipped.
  • Highlighting: Anachronisms are highlighted in the text with a distinct style
  • Modal Details: Click on an anachronism in the sidebar to see details and suggested alternatives
  • Tab Key: Press Tab to swap a highlighted anachronism with a suggested alternative
  • Definitions: Anachronisms include definitions and example phrases from the dictionary
Note: The ★ marker indicates a term is used in previous or related works, which may be appropriate for the setting.

Category Management

Organize your translation work into categories for efficient workflow management.

Figure 7: Category Selector

Category Types:

  • Manual Translation: Items you're translating manually
  • Batch Categories: Auto-generated categories from batch scans

Category Features:

  • Auto-selection: The editor automatically selects the most relevant category
  • Queue Preservation: Items are preserved when switching between categories
  • Batch Integration: Batch scan results create their own categories

Search & Database

Search across your entire project to find specific dialogue lines or terms.

Figure 8: Search & Database Interface

Search Features:

  • Field Search: Search for specific words or phrases in translation, source, Speaker (col 8), Entry Type (col 9), All Text Fields, or a custom field index (select "FIELD_INDEX" to open a modal where you enter the column number)
  • Custom Field Index: When you select "FIELD_INDEX", a modal popup appears asking for the column number (0-based). Enter the CSV column index you want to search. The dropdown will update to show the selected column number (e.g., "FIELD_INDEX (5)").
  • Results Management: Open search results in the editor for editing
  • Persistence: Search results are saved and restored on application restart
Tip: Search results persist across application restarts, so you can continue where you left off.

Keyboard Shortcuts

Use these keyboard shortcuts to work more efficiently (only active in the Editor tab):

  • Ctrl+Enter: Save translation to history
  • Ctrl+ArrowRight: Move to next item
  • Ctrl+ArrowLeft: Move to previous item
  • Ctrl+R: Rewrap editor text
  • Ctrl+D: Replace dashes with em-dash (—)
  • Ctrl+Shift+D: Replace dashes with ellipsis (...)

Troubleshooting

Dependencies Not Installing

If dependencies fail to install automatically, you can install them manually:

pip install -r requirements.txt

Console Encoding Errors

The application automatically sets UTF-8 console encoding on Windows to handle Japanese characters. If you encounter encoding issues, ensure your terminal supports UTF-8.

Cache Issues

The application uses several caching mechanisms to improve performance. Most caches are automatically invalidated when data changes:

  • CSV Files: Automatically reloaded when modified externally (by other contributors or manual text editing). Files written by the application itself are not invalidated to avoid false positives.
  • Glossary Files (bible.csv, glossary.csv): Gloss cache is automatically cleared when these files are modified, ensuring the gloss window displays current lore context.
  • API Cache (config/<lang>/cache.json): Automatically expires after 7 days to ensure fresh API responses. Stored per-language.
  • Prefetch Cache (config/<lang>/prefetch_cache.json): Automatically expires after 7 days. Stored per-language.
  • Queue Caches (config/<lang>/review_queues_cache.json, config/<lang>/review_items_cache.json): Review queues and manual translation queue. Stored per-language, not synced to GitHub.

If you experience persistent cache-related issues, you can manually clear caches by deleting the relevant cache files (config/<lang>/cache.json, config/<lang>/prefetch_cache.json, config/<lang>/review_queues_cache.json, config/<lang>/review_items_cache.json) and restarting the application. Caches will be rebuilt automatically.
